Height and depth of a binary tree


Depth- The depth of a node is the number of edges from the node to the tree's root node.

Finding depth of a tree

Recursively calculate height of left and right subtrees of a node and assign height to the node as max of the heights of two children plus 1. See below pseudo code and program for details.

Note The depth of the root is 0.

Finding Height of a tree

The height of a binary tree is found using the recursive Depth-First Search (DFS) algorithm, as shown below:

  1. Base case: If there is no node, return 0.
  2. Else: If there are 1 or 2 children, return the maximum of the height of the left and right sub-trees, plus 1 to account for the current node. The height of a sub-tree is found by recursively calling the function, and passing the child node as the parameter.

Example

When looking at height:

  1. Every node has height. So B can have height, so does A, C and D.
  2. Leaf cannot have height as there will be no path starting from a leaf.
  3. It is the longest path from the node to a leaf. So A's height is the number of edges of the path to E, NOT to G. And its height is 3.
